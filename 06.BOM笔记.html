浏览器对象模型

三大家族其一 Offset 家族

家族成员： offsetWidth offsetHeight offsetLeft offsetTop offsetParent

offsetWidth offsetHeight （检测盒子自身宽高）

  1. 这两个属性，他们绑定在了所有的节点元素上。获取之后，只要调用这两个属性，我们就能够获取元素节点的宽和高。

  2. offset宽/高 = 盒子自身的宽/高(width/height) + padding +border

offsetLeft 和 offsetTop （检测距离有定位的父盒子的左/上面的距离）

  1. 如果父级都没有定位则以 body 为准, offsetLeft 从父亲的 padding 开始算,父亲的 border 不算。
  2. 在父盒子有定位的情况下，offsetLeft == style.left(去掉 px)

offsetTop/Left 和 style.top/left 的区别：

  1. 最大区别在于 offsetTop/Left 可以返回没有定位的盒子距离左侧的距离。而 style.top/left 不可以

  2. offsetTop/Left 返回的是整数(数值型)，而 style.top/left 返回的是字符串，除了数字外还带有单位：px

  3. offsetTop/Left 只读，而 style.top/left 可读写。（只读是获取值，可写是赋值）

  4. obj.style.xxx 只能获取行内样式

offsetParent （检测父系盒子中带有定位的父盒子节点）

  1. 返回该对象的父级 （带有定位）
      如果当前元素的父级元素没有进行 CSS 定位(absolute,relative,fixed) ,则offsetParent 为 body

  2. 如果当前元素的父级元素中有 CSS 定位 offsetParent 取最近的那个父级元素。

事件对象 Event

  1. Event 对象代表事件的状态

  2. 获得event对象兼容性写法 event || window.event

  3. target与currentTarget

  • event.target 触发事件的目标 获得target兼容型写法 event.target || event.srcElement
  • event.currentTarget 绑定事件的目标

  1. e.pageX/Y 获取鼠标点击的地方相对于页面的位置
    e.clientX/Y 获取鼠标点击的地方相对于可视区域的位置
    e.screenX/Y 获取鼠标点击的地方相对于屏幕的位置
    e.offsetX/Y 表示鼠标指针位置相对于触发事件对象的位置

  2. event.type 事件的类型

  3. event.key 键盘按键码

  4. event.button 鼠标点击的按键(只认识三个键) 可在 onmousedown 事件中测试

    • === 0 您点击了鼠标左键
    • === 1 您点击了鼠标中键
    • === 2 您点击了鼠标右键

事件委托

  1. 通过监听一个父元素，来给不同的子元素绑定事件，减少监听次数，从而提升速度。
  2. 由于事件的冒泡机制,可以使用事件委托的方式给元素添加事件,多用于ul监听事件更改li的情景

阻止冒泡和阻止默认事件

  1. 阻止冒泡
  • event.stopPropagation(); 存在兼容问题 (面试题)
  • IE <= 10 专用 event.cancelBubble = true
  • 兼容写法 event.stopPropagation?event.stopPropagation():event.cancelBubble = true;
  1. 阻止默认事件:

  • 一般有两种: 阻止a标签链接跳转, 阻止form表单提交
    event.preventDefault() || event.returnValue = false

  • 兼容写法:event.preventDefault?event.preventDefault():event.returnValue = false;

Scroll 家族

家族成员: scrollWidth , scrollHeight , scrollTop , scrollLeft

scrollWidth 和 scrollHeight

  • 检测盒子的宽高
  • 内容未溢出时,值=盒子宽高+padding
  • 内容溢出时 值=内容宽高+padding
    • 无滚动条, padding 只显示 上面的和左面的
    • 有滚动条时, 左右的padding只显示一个 ,上下的padding显示两个

scrollTop , scrollLeft 可读写的

  1. 前提: 目标元素有滚动条
  2. 被浏览器或父类遮挡的头部和左边部分。
  3. 可以获取或设置一个元素的内容垂直滚动或水平滚动的像素数。element.scrollTop/Left = XXX

获取页面卷入的高度

兼容写法：
var scrollTop = document.documentElement.scrollTop || window.pageYOffset || document.body.scrollTop

onscroll 事件

  • 当元素的滚动条滚动时触发的事件。

  • onscroll事件貌似任何实体元素都可以绑定，这里的实体元素包括DOM元素、window元素、document元素。

  • 用法：element.onscroll=function(){};

  • 注意: 设置此事件的元素一定要有滚动条

关于window.scroll(),window.scrollBy(),window.scrollTo() 不兼容IE

  • window.scroll(x,y)是让window滚动条滚动到(x,y)坐标。

  • window.scrollBy(-x,-y)是让window滚动条相对滚动到某个坐标，- 10即相对向左/向上滚动10像素。

  • window.scrollTo(x,y)和window.scroll(x,y)一样。

  • element.scrollIntoView(boolean) 让元素贴顶或者贴底，相对于可视区域

    • true 贴顶
    • false 贴底

Client 家族

家族成员 clientWidth clientHeight clientTop clientLeft

clientWidth clientHeight

  1. clientWidth clientHeight

    • 检测盒子的宽高
    • 自身宽高 + padding
    • 内容溢出不算,滚动条(默认宽度17)不算

  2. document.documentElement.clientWidth/clientHeight

    • 获取浏览器可视区域的宽高
    • 不计算滚动条 没有兼容问题

  3. window.innerWidth/innerHeight

    • 表示获取 window 可视区域的内部大小(带滚动条)
    • IE <= 8 不支持

  4. window.outerWidth/outerHeight

    • 表示整个浏览器窗体的大小
    • IE <= 8 不支持

clientTop clientLeft 只读

  • 一个元素顶部和左侧边框(border)的宽度

获取元素的样式

  1. 行内样式可以通过 ele.style.styleName 获取

  2. 内联样式和外联样式可以通过以下方式获取

/**
* 兼容写法,获取任意元素的css样式
**/
function getStyle(ele,styleName){
	if(ele.currentStyle){
		return ele.currentStyle[styleName];
	}else{
		return window.getComputedStyle(ele,null)[styleName];
	}
}
